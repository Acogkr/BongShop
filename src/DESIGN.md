◈ 서버환경: Paper 1.21.4+
◈ 플러그인 장르: 상점 시스템
◈ 플러그인 이름: BongShop

◈ 요구사항
- 플러그인에서 뜨는 모든 숫자값은 #,### 포맷
- placeholder 연동
- vault 연동
- coinsengine 연동
- nexo 아이템 연동
- itemsadder 아이템 연동
- 메시지, 사운드, GUI 설정 커스텀 가능

◈ 플러그인 시스템 요약
- 저번에 맡긴 일일상점 플러그인이랑 어느정도 비슷하게 해주시면 됩니다.
- 유저가 상점 GUI에서 아이템을 구매/판매하는 시스템입니다.
- 아이템마다 시세가 있으며 특정 시간마다 시세가 변경됩니다.
- 아이템에 재고를 설정할 수 있습니다. 재고는 특정 시간대에 입고됩니다.
- 아이템에 구매 제한을 설정할 수 있습니다. 특정 아이템을 제한 갯수만큼 구매를 하면 더 이상 구매할 수 없습니다.

◈ 플레이스홀더
%bongshop.playerselltotal.all% : 모든 상점에서 해당 플레이어가 판매하여 번 금액을 보여줍니다. 예: 10,000
%bongshop.playerselltotal.<shop.id>% : <shop.id>상점에서 해당 플레이어가 판매하여 번 금액을 보여줍니다. 예: 50,000
%pricechange.time.second% : 가격변동까지 남은 초. 예: 40
%pricechange.time.minute% : 가격변동까지 남은 분. 예: 20
%pricechange.time.hour% : 가격변동까지 남은 시. 예: 1

◈ 관리자, 콘솔 명령어
/상점 생성 <유형> <상점이름> <상점아이디>: 상점을 생성합니다. 유형은 구매,판매가 있습니다. #예: /상점 생성 구매 치장상점 cosmetic
/상점 열기 <상점아이디> <player>: <player>에게 <상점id>의 상점을 엽니다.
/상점 삭제 <상점아이디>: <상점아이디>의 상점을 삭제합니다.
/상점 판매기록 <플레이어>: 해당 플레이어의 판매기록을 채팅으로 출력합니다. (메시지 설정 참고)

◈ 콘피그 설정
가격 변동 시간 : 가격 변동 시간 간격 / 분단위 예 60 = 60분
재고 입고, 판매 횟수 초기화 시간 / 예 00:00

◈ 메시지, 로어 설정 #제가 미처 적지 못한 메시지 설정이 있을 수 있습니다.
구매 로어:
- vault로어:
    - "<white>구매 가격: [price] 골드"
- coinsengine로어:
    - "<white>구매 가격: [price] <coin_name>"
- 아이템로어:
    - "<white>구매 가격: <itemname> [price]개"
- 전체 재고가 있는 아이템 로어:
    - 기본: "<white>남은 재고 : [amount]/[max_amount]"
    - 남은 재고 없으면: "<whte> 남은 재고 : <red>재고 없음"
- 구매 제한이 있는 아이템 로어:
    - 기본: "<white>구매 제한 갯수 : [amount]/[max_amount]"
    - 남은 갯수 없으면: "<whte> 구매 제한 갯수 : <red>구매 불가"
      판매 로어:
- 하루당 판매 횟수가 있는 아이템 로어:
    - 기본: "<white>하루당 판매 갯수 : [amount]/[max_amount]"
    - 남은 갯수 없으면: "<whte> 남은 판매 갯수 : <red>판매 불가"

- vault 재화가 부족할 때: "<red>총 [구매할 금액-보유 금액] 골드 만큼 부족하여 아이템 구매가 불가능합니다."
- coinsengine 재화가 부족할 때: "<red>총 [구매할 금액-보유 금액] <coinsengine> 만큼 부족하여 아이템 구매가 불가능합니다."
- 아이템 재화가 부족할 때: "<red>총 [구매할 수량-보유 수량] <coinsengine> 만큼 부족하여 아이템 구매가 불가능합니다."
- 구매했을 때: "<green>[item]을 [price]원에 [amount]개 구매했습니다."
- 판매했을 때: "<red>[item]을 [price]원에 [amount]개 판매했습니다.
- 판매할 아이템이 없을 때: "<red>충분한 아이템을 가지고 있지 않습니다."
- 재고가 없을 때: "<red>해당 아이템은 품절되었습니다."
- 구매 제한상태일 때: "<red>이 아이템의 최대 구매 갯수를 초과하였습니다."
- 인벤토리 공간이 없을 때: "<red>인벤토리 공간이 없어 구매가 취소되었습니다."
- [전체메시지] 가격이 변동되었을 때: "<gold>상점 가격이 업데이트 되었습니다!"
- [콘솔로그] 유저가 아이템을 구매했을 때: "BongShopLog >> displayname(playername)님이 [상점이름]에서 [item]을 [price]원에 [amount]개 구매했습니다."
- [콘솔로그] 유저가 아이템을 판매했을 때: "BongShopLog >> displayname(playername)님이 [상점이름]에서 [item]을 [price]원에 [amount]개 판매했습니다."
- 관리자에게 뜨는 판매기록 메시지:
    - "<white>%player_displayname%님의 판매기록
    - "<white>모든 상점 판매 토탈 : [amount]원
    - "<white><상점이름> 판매 토탈 : [amount]원
    - "<white><상점이름> 판매 토탈 : [amount]원
    - "<white><상점이름> 판매 토탈 : [amount]원

◈ 유저가 아이템을 판매할 수 있는 판매상점 파일 설정
좌클릭: 1개 판매
쉬프트 좌클릭: 64개 판매
7번 키: 모두 판매

GUI 이름
GUI 줄 수 설정: 1~6
가격변동타입: 랜덤으로 가격 변동 / 최근 판매량 10건에 따라 가격 변동(많이 팔릴 시 가격 ↓ / 적게 팔릴 시 가격 ↑)
다음, 이전 페이지 버튼 / 이름, 로어, 슬롯, 커모데 변경 가능
변동까지 남은 시간 버튼 / 이름, 로어, 슬롯, 커모데 변경 가능(남은 시간 플레이스홀더 필요)
아이템 설정:
- 판매아이템: 판매할 아이템 / itemsadder,nexo 아이템 가능
- 판매아이템 수량 / 예시로 판매아이템 감자에 수량 30, 가격 5일 때, 감자 30개를 5원에 판매하는 방식
- 상점에 표시할 아이템 이름 / 설정안할시 기존 아이템 이름
- 아이템에 추가로 표시할 로어 / 판매 아이템의 기존 로어 밑에 추가되는 방식
- 판매 정가 가격 / 예: 1500
- 판매 변동 가격 / 예: 1000~2000
- 판매아이템 슬롯 설정
- 하루당 판매할 수 있는 갯수 / 설정 안할 시 무제한

◈ 유저가 아이템을 구매할 수 있는 구매상점 GUI 설정
좌클릭: 1개 구매
쉬프트 좌클릭: 64개 구매
7번 키: 16개 구매

GUI 이름
GUI 줄 수 설정: 1~6
가격변동타입: 랜덤으로 가격 변동 / 최근 판매량 10건에 따라 가격 변동(많이 팔릴 시 가격 ↓ / 적게 팔릴 시 가격 ↑)
다음, 이전 페이지 버튼: 이름, 로어, 슬롯, 커모데 변경 가능
변동까지 남은 시간 버튼: 이름, 로어, 슬롯, 커모데 변경 가능 / 남은 시간 플레이스홀더 필요
아이템 설정:
- 구매아이템 / itemsadder/nexo 아이템 가능
- 상점에 표시할 아이템 이름 / 설정 안할 시 기존 아이템 이름
- 아이템에 추가로 표시할 로어 / 판매 아이템의 기존 로어 밑에 추가되는 방식
- 구매 재화 / vault, coinsengine, item
- 구매 정가 가격 / 변동 기준이 되는 기본 가격 (예시) 1500 / 재화가 아이템이면 수량
- 구매 변동 가격 / 변동되는 범위 (예시) 1000~2000 / 설정안할 시 변동x
- 구매아이템 슬롯 설정
- 모든 유저가 공유되는 전체 재고 / 설정 안할 시 무제한
- 하루당 구매할 수 있는 갯수 / 설정 안할 시 무제한
- 구매 제한 갯수(아이템이 변경되지 않는 이상 구매 못함) / 설정 안할 시 무제한
